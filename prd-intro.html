什么是 PRD？

小P带你认识产品经理的「秘密武器」——PRD（产品需求文档）

它就像计划书、说明书、菜谱……我们一起来看看！

PRD 是什么？🤔

PRD 就是一份「产品说明书」，告诉别人你要做什么产品，为什么做，怎么做。

🍰

像菜谱

做菜前写好食材、步骤，别人才能跟着做。

🗺️

像地图

出门前查好路线，才知道每一步往哪走。

📐

像图纸

盖房子前画好图，工人知道门开在哪。

🏠

没有 PRD 会怎样？

❌ 没有 PRD

随便盖的房子

  • 工人以为厨房在二楼
  • 盖完发现没有洗手间
  • 门开在了天花板上
  • 大家互相责怪
✅ 有 PRD

有图纸的房子

  • 每个人都知道目标
  • 房间尺寸清清楚楚
  • 材料时间好安排
  • 出错少，效率高
🎂

再做一道题：生日派对

❌ 没有 PRD

临时抱佛脚

  • 忘了邀请好朋友
  • 蛋糕买成榴莲味
  • 场地订错日期
  • 游戏环节一团乱
✅ 有 PRD

提前写计划

  • 时间、地点、人员列好
  • 提前问朋友喜欢吃什么
  • 游戏流程安排明白
  • 派对顺顺利利
🎮

做网页应用也是同理

如果你想做一个「班级运动打卡网页」：

🎯

目标

全班同学每天记录运动，一个月后看谁最坚持。

👤

用户

小学五年级学生、体育老师、班主任。

⚙️

功能

打卡、排行榜、提醒、数据统计。

🔄

流程

打开网页 → 登录 → 选择运动 → 提交打卡。

为什么要做 PRD？🌟

🧠

想清楚

逼自己想明白：解决什么问题？谁会用？

🗣️

讲清楚

让队友、老师、程序都能理解你要做什么。

🚀

少返工

提前发现问题，避免做了一半方向错了。

PRD 有哪几个部分？📝

一个最简单的 PRD，通常包含下面四块内容：

🎯

背景与目标

为什么要做？解决什么问题？成功标准是什么？

👤

用户与场景

谁会用？在什么时候、什么地方用？

⚙️

功能需求

产品能做什么？哪些最重要？

🔄

流程与交互

用户怎么一步步操作？页面长什么样？

🛠️

今天我们要做什么？

我们要一起完成一份属于你自己的 PRD！

选择一个感兴趣的方向：运动、环保、学习、健康饮食……
然后一步步定义产品，最后还能生成技术方案。
